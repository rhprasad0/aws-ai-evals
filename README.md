# AWS AI Evals

This repo is being reset.

The first attempt built a candidate-evidence chatbot and an AWS/Bedrock eval harness around it. That work was useful as a learning spike, but the schema design and eval shape accumulated too much cognitive debt. Rather than keep layering tools on a shaky contract, the repo is being rolled back to a minimal starting point.

Lesson from attempt one: design the evaluation schema, labeling workflow, and success criteria before building the harness around them.

Next direction comes from Ryan. Agents should not revive the previous roadmap, chatbot assumptions, rubrics, datasets, or infrastructure without explicit instruction.
